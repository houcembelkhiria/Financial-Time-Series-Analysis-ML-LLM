## How Can I Contribute?

### Reporting Bugs

If you find a bug, please create an issue with:
- A clear title and description
- Steps to reproduce the issue
- Expected vs actual behavior
- Your environment (R version, Python version, OS)
- Screenshots if applicable

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please include:
- A clear title and description
- Why this enhancement would be useful
- Possible implementation approach

### Pull Requests

1. Fork the repo and create your branch from `main`
2. If you've added code, add tests if applicable
3. Ensure your code follows the existing style
4. Write a clear commit message
5. Update documentation as needed

## Code Style

### R Code
- Use tidyverse conventions
- Comment complex operations
- Use meaningful variable names
- Keep functions focused and modular

### Python Code
- Follow PEP 8
- Use type hints where appropriate
- Document functions with docstrings
- Keep functions under 50 lines when possible

## Development Setup

```bash
# Clone your fork
git clone https://github.com/your-username/Financial-Time-Series-Analysis-ML-LLM.git

# Create a branch
git checkout -b feature/your-feature-name

# Make your changes
# ...

# Commit
git commit -m "Add: Brief description of your changes"

# Push
git push origin feature/your-feature-name
```

## Testing

Before submitting a PR:
- [ ] Test that the notebook runs end-to-end
- [ ] Verify all visualizations render correctly
- [ ] Check that models train successfully
- [ ] Ensure no new linter errors

## Documentation

- Update README.md if you change functionality
- Add comments to complex code sections
- Update the user guide if adding features

## Questions?

Feel free to open an issue with the "question" label.
